TestNG Listeners and Extent Reports (day46)

Overview
- Purpose: Implement test listeners for custom reporting and generate detailed HTML reports.
- Files: ListenerClassDemo.java, ListenerClassDemo2WithoutXML.java, MyListener.java, ReportManagerExtent.java, testng.xml, extenttestng.xml

TestNG Listeners
- ITestListener: Interface for test lifecycle events (onStart, onTestStart, onTestSuccess, onTestFailure, onTestSkipped, onFinish)
- ISuiteListener: Interface for suite-level events
- IInvokedMethodListener: Interface for method invocation events
- Implementation: Create class implementing listener interface, register in testng.xml or via @Listeners annotation

Extent Reports
- Third-party reporting library for beautiful HTML reports
- Features: Screenshots, test categorization, timeline view, dashboard
- Setup: ExtentReports instance, ExtentTest for individual tests, attach screenshots on failure

Examples Explained
1) MyListener.java
- Implements ITestListener with custom logic for each test event
- Logs test status, captures screenshots on failure, generates custom reports

2) ListenerClassDemo.java + ListenerClassDemo2WithoutXML.java
- Demonstrates listener registration via XML and @Listeners annotation
- Shows different approaches for listener implementation

3) ReportManagerExtent.java
- ExtentReports setup and configuration
- Screenshot capture and attachment to reports
- Report generation and cleanup

4) testng.xml / extenttestng.xml
- XML configuration for listener registration
- Suite-level listener setup for comprehensive reporting

Good Practices
- Use listeners for cross-cutting concerns (logging, screenshots, reporting)
- Keep listener logic lightweight to avoid performance impact
- Generate reports in separate directory with timestamps
- Clean up old reports to manage disk space
